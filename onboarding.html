Deploy the TurboSanta API

Must be completed first — everything else depends on this

The Apps Script API is the invisible brain connecting your Sheet to the live map, Santa's speed calculations, and all your public tools.

  1. In your Sheet, go to Extensions → Apps Script.

  2. Click Deploy → New deployment.

  3. Choose Web app as the deployment type.

  4. Set Execute as: Me and Who has access: Anyone.

  5. Click Deploy, authorise permissions, and copy the Web App URL.

  6. Paste the URL into the Settings tab — find the row where Column A = api_url and set Column B to your URL.

⚠️
Always update deployments after edits

If you ever edit Apps Script, go to Deploy → Manage deployments → Update. Old deployments serve stale code and will cause silent bugs.

Firebase Setup

Live sleigh location, analytics, and subscriber notifications

Understanding Your Two Firebase Keys

TurboSanta uses two distinct keys in the Tracker sheet. These are not interchangeable:

KeyWhat it doesWhere it lives
secret GPS write key. Embedded in the GPS Logger app URL. Permits writing the live location only. Safe to rotate if the sleigh phone is lost. GPS Logger app URL
firebase_db_secret Admin key (Legacy Database Secret). Used server-side by Apps Script for reading subscribers, analytics, and history. Never exposed to the public. Apps Script only
⚠️
Common Mistake

Setting both keys to the same value appears to work but breaks your Firebase security rules. The secret is used as a path segment in Firebase rules; firebase_db_secret is used for authentication. They must be different.

Create Your Firebase Project

  1. Visit https://firebase.google.com and create a new project.

  2. Go to Build → Realtime Database and create a database — choose Region: Europe.

  3. Copy your Database URL and paste it into Tracker → firebase_url.

  4. Go to Project Settings → Service Accounts → Database Secrets to find your firebase_db_secret.

Create Placeholder Nodes

Firebase automatically deletes empty nodes. The following placeholders must exist before any data is sent — create each with the value placeholder: true:

  • location
  • location/current
  • analytics
  • analytics/views
  • analytics/waves

Security Rules

Open Realtime Database → Rules and paste the following. Replace YOUR_SECRET with your actual secret value.

JSON Firebase Security Rules 
{
  "rules": {
    ".read": false,
    ".write": false,
    "location": {
      "current": {
        // 🔓 ALLOW PUBLIC READ: So the frontend tracker can fetch GPS directly!
        ".read": true,
        "$secret": {
          ".write": "$secret === 'YOUR_SECRET'"
        }
      },
      "history": {
        ".read": false,
        ".write": "auth !== null",
        ".indexOn": ["ts"]
      }
    },
    "analytics": {
      ".read": false,
      ".write": false,
      "views": { ".indexOn": ["ts"] },
      "waves": { ".indexOn": ["ts"] }
    },
    "subscribers": {
      ".read": false,
      ".write": false
    }
  }
}

Automatic Data Cleanup

TurboSanta prunes old Firebase data automatically in the background to keep things lean:

Data TypeRetention Rule
GPS HistoryEntries older than 90 minutes are deleted every time the notification checker runs.
Wave EventsEntries older than 12 hours are deleted. God Mode wave counts reflect tonight's activity only.
SubscribersNever-notified subscribers deleted after 24 hours. Notified subscribers deleted 6 hours after the "Santa has passed" email.

Advanced Settings

Key settings rows not covered by the Wizards — all live in the Settings tab
ℹ️
How the Settings Tab Works

The Settings tab has three separate data areas, not just one. Column A/B is the key-value store. Column Q and Column S hold additional list data used by specific features — do not delete or move these columns.

PA Presets (Column S)

Column S of the Settings sheet (from S2 downwards) holds pre-written PA announcement messages. The API reads this column on every request and returns them as a presets array, which God Mode loads into a dropdown so your Lead Elf can fire canned announcements quickly without typing on a cold night.

  • Each cell in S2:S is one preset message — one per row.
  • Blank cells between entries are ignored.
  • These are separate from the public PA system's live message — presets are pre-loaded options; live_announcement is what's actively broadcasting.
  • Examples: "Santa is running 10 minutes late — thank you for your patience!" or "Challenge: if we raise £50 in the next 5 minutes, the elves will do a dance!"

Donations Route Dropdown (Column Q)

Column Q of the Settings sheet (from Q2 downwards) holds the list of route names that appear in the Donations Wizard dropdown when logging cash/card totals. Keep this in sync with your Routes tab.

Live Announcement (PA System)

The live_announcement key in Settings is the public PA system. When set, its value overrides the normal status box on every viewer's tracker with a pulsing gold announcement banner.

  • Set via the God Mode Dashboard PA panel on sleigh night.
  • Clear the value to return to normal ETA/status messages.
  • Examples: "Santa is running 10 minutes late — thank you for your patience!" or "Challenge: raise £50 in the next 5 minutes and the elves will do a dance!"

Crew Alert

The crew_alert key in Settings drives internal messaging from the Lead Elf to all volunteers on the Crew Hub — completely separate from the public PA system and never visible on the public tracker.

PropertyDetail
Default valueLeave blank
Written byLead Elf via God Mode on sleigh night
Visible toAll volunteers with Crew Hub open, including the QR donation screen
Poll frequencyEvery 15 seconds
💡
How it works

Lead Elf types a message in the Crew Alert panel in God Mode → hits Send → Apps Script writes to the crew_alert cell → within 15 seconds, a pulsing red banner appears above the tabs on every volunteer's Crew Hub screen, including full-screen QR donation views.

Elf Cam URL

The elf_cam_url key powers the Elf Dashboard polaroid feature. When the Elf Cam captures a new photo, it writes the Google Drive URL here. The tracker polls for changes and displays any new image as a floating Polaroid on the public map.

  • Written automatically by the Elf Cam capture system — no manual editing needed.
  • Google Drive URLs are automatically converted to direct thumbnail links by the tracker.
  • Polaroids only appear on tabs that are currently visible (battery-aware).

GET_PEAK_VIEWERS Formula

TurboSanta includes a custom Google Sheets formula to calculate peak simultaneous viewers within any time window:

Google Sheets Custom Formula 
=GET_PEAK_VIEWERS(E2:E, 15)
ArgumentDescription
Argument 1A range of timestamps — the E column in Tracker Analytics (parsed DateTime values).
Argument 2Window size in minutes. 15 is a sensible default; reduce to 5 for a tighter peak calculation.
ReturnsThe highest number of viewers active within any rolling window of that duration.

Subscriber Email Notifications

Up to two automated emails per subscriber per sleigh night

Email 1 — "Santa's Nearly Here"

Triggered when either condition is met and GPS is fresh (under 90 seconds old):

  • Santa's estimated arrival is under 5 minutes based on calculated speed, or
  • Santa is within 250 metres of the subscriber's pin.

The email is sent once and the subscriber is flagged as notified. If no email address is stored, the flag is still set for internal tracking.

Email 2 — "Santa's Just Been Past"

Triggered after Email 1 when either condition is met:

  • Santa has moved more than 150m further away from the subscriber's closest recorded distance AND at least 1 minute has passed since Email 1, or
  • 20 minutes have passed since Email 1 (timeout fallback).

After this email, the subscriber record is scheduled for deletion 6 hours later.

Testing Without a Live Sleigh

  1. Temporarily set a test Firebase location manually in the Firebase console (location/current) with a lat/lng close to a test subscriber address.

  2. Create a test subscriber record in Firebase under /subscribers with a valid email, lat, lng, and notified: false.

  3. Open Apps Script → Run checkSubscribersAndNotify manually.

  4. Check the Apps Script execution log for trigger messages and verify the email arrives.

⚠️
Month Guard — Currently Disabled

The notification code contains a month guard that is currently force-overridden to allowEmails = true for testing. Before going live, locate this line in checkSubscribersAndNotify and switch it to the commented-out production version: const allowEmails = (month === 11 || month === 0); — this restricts emails to December and January only.

MP4 Generator

Animated route videos for social media, WhatsApp, and sponsor updates

One-Time GitHub Token Setup

TurboSanta triggers rendering via a secure GitHub Actions workflow. This requires a Personal Access Token stored in your Apps Script project:

  1. Go to github.com → Settings → Developer settings → Personal access tokens.

  2. Create a Classic token with scopes: repo and workflow.

  3. Copy the token.

  4. In your Sheet: Extensions → Apps Script → Project Settings → Script Properties.

  5. Add a property with key GITHUB_TOKEN and paste your token as the value.

  6. Save — the render button in the MP4 Generator wizard will now activate.

🔒
Security Note

Your token is stored in Script Properties and is never exposed to the public or included in API responses. If you suspect compromise, revoke it on GitHub and create a new one.

What Happens After You Click Render

  • A GitHub Actions workflow is triggered with your route name, aspect ratio, and API URL.
  • The Routes sheet mp4Status column updates to 🎥 Rendering.
  • When complete, the workflow posts back to your API and the mp4Url column is updated automatically.
  • No polling or manual refresh needed — it updates entirely in the background.

Viral Protection & Scaling

Optional — for power users expecting large audiences

Google Apps Script has a hard limit of 30 simultaneous connections. If your tracker link goes viral on a local Facebook group, hundreds of simultaneous visitors can crash the tracker. A free Cloudflare cache eliminates this risk entirely.

⚡
How it works

Cloudflare acts as a shield — it sits in front of your Apps Script backend and serves cached configurations directly to users. Combined with our new direct Firebase GPS connection, your Google Workspace quotas are fully protected. Cost: £0.00. Capacity: 100,000+ simultaneous viewers.

Step-by-Step Cloudflare Setup

  1. Visit https://dash.cloudflare.com and create a free account.

  2. Go to Workers & Pages → Create Worker. Name it (e.g., santa-api), click Deploy, then click Edit Code.

  3. Delete all default code and paste the worker script below.

  4. Critical: Replace the GAS_URL on line 3 with your actual Google Apps Script Web App URL.

  5. Click Deploy. Cloudflare gives you a new Worker URL (e.g., https://santa-api.your-account.workers.dev).

  6. In your Google Sheet, find the api_url row in the Settings tab and replace Column B with your new Cloudflare Worker URL.

JavaScript Cloudflare Worker — paste into editor 
export default {
  async fetch(request, env, ctx) {
    // ✏️  Replace with your Google Apps Script Web App URL:
    const GAS_URL = "https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec";

    const url = new URL(request.url);
    const targetUrl = GAS_URL + url.search;

    const cache = caches.default;
    const cacheKey = new Request(targetUrl, request);
    let response = await cache.match(cacheKey);

    if (!response) {
      response = await fetch(targetUrl);
      response = new Response(response.body, response);

      // Cache for exactly 5 seconds
      response.headers.set("Cache-Control", "s-maxage=5");
      response.headers.set("Access-Control-Allow-Origin", "*");

      ctx.waitUntil(cache.put(cacheKey, response.clone()));
    } else {
      response = new Response(response.body, response);
      response.headers.set("Access-Control-Allow-Origin", "*");
    }

    return response;
  }
};

Sat-Nav Driver Mode

Precise GPX-following navigation for the towing vehicle

TurboSanta's Sat-Nav strictly follows your loaded GPX routes — unlike Google Maps, which tries to recalculate and skip cul-de-sacs. On sleigh night, you'll run two separate devices:

Device A
The Beacon (Public GPS)
Runs the GPS Logger app in the background, pushing Santa's location to the public map. Turn it on, plug it into a power bank, mount it firmly — then leave it alone for the entire route.
Operated by: Nobody
Device B
The Navigator (Sat-Nav)
Reads the phone's local GPS to display a blue directional arrow, keeping the car perfectly centred on the red route line. Keeps the screen awake automatically.
Operated by: Lead Navigator (passenger) or mounted on dashboard

Activating Sat-Nav Mode on the Night

  1. Open your Dab Santa Landing Page and click Sat Nav. (Alternatively, open your standard tracker link and add &driver=1 to the end of the URL.)

  2. A large red 🧭 START SAT-NAV MODE button will appear. Tap it.

  3. If your browser requests Location Permissions, tap Allow.

  4. The screen zooms to street level, the blue arrow appears, and public UI buttons (Donate, Notify) vanish to prevent distractions.

  5. Drive the red line. When the route is done, close the browser tab to stop GPS and save battery.
